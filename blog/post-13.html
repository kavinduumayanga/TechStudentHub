SEO Basics

Simple SEO for a Student Website (No Spam)

SEO doesn’t need tricks. If you’re a student building a site, the best SEO is a clean structure, helpful content, and basic technical hygiene. This guide covers the simple steps that matter for a static site—without spam tactics.

What SEO is (for students)

SEO (Search Engine Optimization) is the process of making it easier for search engines to understand your site and match your pages to relevant searches. It’s not about gaming algorithms. It’s about:

  • Clear content structure
  • Helpful pages that answer questions
  • Technical basics (links, metadata, crawlability)

Titles and headings

Your <title> tag is one of the strongest signals about a page’s topic. Keep it:

  • Specific and descriptive
  • Not stuffed with keywords
  • Unique per page

Headings (h1, h2, h3) should match your content sections. One h1 per page is a good default.

Meta descriptions

Meta descriptions don’t directly “rank” your site in a simple way, but they influence click-through because they show in search results. Write descriptions that:

  • Explain what the page gives the reader
  • Are different per page
  • Do not promise something the page doesn’t deliver

Internal linking (the underrated part)

Internal links help users navigate and help search engines discover your pages. A good internal linking strategy for a student site:

  • Keep navigation consistent across pages
  • Link related blog posts to each other
  • Use descriptive anchor text (not “click here”)

For example, if someone reads about learning web dev, link them to practical guides like getting started and building a first static site.

Sitemaps and robots.txt

A sitemap tells search engines which pages exist. A robots.txt file gives crawl guidance and can point at the sitemap location.

On small static sites, these files are easy wins because they prevent “missing page discovery” issues when you add new posts.

Content quality signals

For long-term results, content quality matters more than “SEO tricks.” Quality signals include:

  • Clear writing and helpful explanations
  • Original structure (not copied templates)
  • Accurate, consistent information
  • Functional links and images
  • Fast loading and mobile-friendly layout

A quick SEO checklist

  • Unique <title> and description per page
  • Headings in order (h1 → h2 → h3)
  • Internal links are correct (no broken paths)
  • Images have alt text
  • Sitemap includes new posts
  • robots.txt points to sitemap

If you host on GitHub Pages as a project site, ensure internal links use correct relative paths. See GitHub Pages hosting explained.
