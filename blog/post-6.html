CSS Fundamentals Explained with Practical Examples

CSS is where your site becomes readable, responsive, and pleasant to use. Beginners often think CSS is “random” because changes don’t behave as expected. The truth is CSS follows rules—once you learn a few fundamentals (selectors, specificity, box model, layout), it becomes predictable.

What CSS does

CSS controls presentation: spacing, colors, fonts, layout, and responsiveness. Your browser reads HTML first, then applies CSS rules to elements that match selectors.

A simple mental model is: select elements → apply rules → resolve conflicts → render boxes.

Selectors (the useful 20%)

Start with the selectors you’ll use every day:

  • element selectors: p, h1, a
  • .class selectors: .button, .card
  • #id selectors: #main (use sparingly)
  • Descendants: .nav a
  • Pseudo-classes: a:hover, button:focus

Example:

.card {
  padding: 16px;
  border: 1px solid #ddd;
}

.card h2 {
  margin-top: 0;
}

a:hover {
  text-decoration: underline;
}

Specificity (why styles don’t apply)

If two rules target the same element/property, CSS uses a priority system called specificity (plus source order). A simplified rule of thumb:

  • Inline styles win over everything (avoid them).
  • #id is stronger than .class.
  • .class is stronger than element selectors.
  • If specificity is equal, the later rule wins.

When you feel stuck, inspect the element in devtools and look at which rule is crossed out.

Box model (margin, border, padding)

Every element is a box. The box model explains spacing:

  • Content: the text/image area
  • Padding: space inside the border
  • Border: the line around the element
  • Margin: space outside the border

Example:

.button {
  padding: 10px 14px;
  border: 1px solid currentColor;
  margin: 8px 0;
}

A common beginner bug is “my width doesn’t fit.” That’s often because padding/border are added on top of width. Using box-sizing: border-box; is a common fix (many sites set it globally).

Layout basics: flex and grid

Modern CSS layout is mostly flexbox and grid. You don’t need to master every property immediately.

Flexbox (one-dimensional layouts)

Use flexbox to align items in a row or a column:

.row {
  display: flex;
  gap: 12px;
  align-items: center;
  justify-content: space-between;
}

Common use cases: navbars, button rows, cards in a row.

Grid (two-dimensional layouts)

Use grid for page layouts and card grids:

.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

Grid becomes powerful when you want rows and columns at the same time.

Responsive design (mobile-friendly)

Responsive design means your page works on phones, tablets, and desktops. Start with these patterns:

  • Use flexible units (%, rem) instead of fixed pixels everywhere.
  • Limit line length for readability.
  • Use media queries for layout changes.

Example media query:

@media (max-width: 720px) {
  .grid {
    grid-template-columns: 1fr;
  }
}

Practical habits that keep CSS manageable

  • Name classes by purpose (e.g., .card, .meta, .cta-row) rather than by appearance.
  • Avoid deep selectors like .page .container .section .card h2 unless needed.
  • Prefer small reusable utilities (spacing, alignment) over repeated inline styles.
  • Test in devtools before editing a lot of CSS.
