HTML Basics Every Beginner Must Know

HTML (HyperText Markup Language) is the structure of the web. When you understand a small set of core tags and a few best practices, you can build pages that are readable, accessible, and easy to style with CSS. This guide covers the HTML basics beginners must know—without overload.

What HTML is (and what it isn’t)

HTML describes the structure of a page: headings, paragraphs, navigation, images, and forms. It does not control spacing, colors, or layout—that’s CSS. It also does not add complex interactivity—that’s usually JavaScript.

Think of HTML as a well-labeled document. When you use the right tags, browsers and assistive technologies (like screen readers) can understand your content properly.

A minimal HTML page skeleton

Here is the smallest “real” HTML page most projects start with:

<!doctype html>
<html lang="en">
<head>
  <meta charset="utf-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1" />
  <title>My Page</title>
</head>
<body>
  <h1>Hello</h1>
  <p>My first page.</p>
</body>
</html>

Three beginner tips:

  • lang helps screen readers and translation tools.
  • The viewport meta tag helps your layout work on mobile.
  • The title shows in the browser tab and search results.

Semantic tags you should actually use

Semantic tags describe meaning. They make your site easier to maintain and improve accessibility.

Common semantic layout tags

  • <header> — top-of-page area (branding, primary nav)
  • <nav> — navigation links
  • <main> — the primary content of the page
  • <article> — a standalone piece of content (like a blog post)
  • <section> — a thematic grouping of content
  • <footer> — the bottom area (legal links, contact)

Use headings in order: one <h1> per page, then <h2> for major sections, and <h3> for sub-sections. This makes your page easy to scan.

Links and images (the correct patterns)

Links

Use <a href="..."> for navigation and references. Good links have clear text:

  • Bad: “Click here”
  • Better: “Read the CSS fundamentals guide”

On GitHub Pages project sites, avoid links that start with /. Use relative links (this site already follows that rule).

Images

Every image should have an alt attribute. If the image is informative, describe it. If it’s decorative, use alt="".

<img src="../assets/images/tech-student-hub-thumbnail.svg"
     alt="Diagram showing a simple HTML page structure"
     width="1200" height="630" />

Width/height help reduce layout shifts while the image loads.

Lists and content grouping

Use lists when you have a set of items. This improves readability and makes your content easier to scan.

  • <ul> for unordered lists (no specific order)
  • <ol> for ordered lists (steps or ranked items)

Group related content with <section> and use <figure> + <figcaption> for images with captions.

Forms: the beginner essentials

Forms collect input. The best beginner habit is pairing every input with a <label>:

<label for="email">Email</label>
<input id="email" name="email" type="email" required />

Basic input types you’ll use often:

  • type="text" for general text
  • type="email" for email addresses
  • type="password" for passwords
  • <textarea> for multi-line messages

This site uses a mailto form on the Contact page (static-friendly). You can study it at Contact.

Accessibility basics that help everyone

You don’t need to memorize every accessibility rule to start. Focus on these beginner-friendly habits:

  • Use semantic tags and proper heading order.
  • Make link text meaningful (“Read our Git guide”).
  • Ensure form fields have labels.
  • Add alt text to images.
  • Test with keyboard-only navigation (Tab/Shift+Tab).

Accessibility is also a quality signal: it makes your site more usable for more people.
